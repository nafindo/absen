## 1️⃣ LAMAN: SPLASH & LOGIN

### Prompt A: Splash Screen

```markdown
## KONTEKS
Buat Splash Screen untuk aplikasi Android native "Absensi Pro — Admin Dashboard" 
menggunakan Jetpack Compose. Aplikasi ini adalah dashboard admin untuk sistem 
absensi karyawan multi-toko berbasis Google Apps Script.

## TUGAS
Buatkan composable `SplashScreen` yang:
1. Menampilkan logo aplikasi (placeholder composable)
2. Menampilkan nama aplikasi "Absensi Pro" dengan subtitle "Admin Dashboard"
3. Menampilkan versi aplikasi "v3.0"
4. Memiliki animasi fade-in + scale-up logo selama 800ms
5. Auto-navigate ke LoginScreen setelah 2 detik menggunakan LaunchedEffect
6. Cek session: jika user sudah login (SharedPreferences), navigate ke Dashboard

## SPESIFIKASI UI
- Background: Gradient dari Color(0xFF1A73E8) ke Color(0xFF1557B0)
- Logo: Box 120dp dengan icon Shield (Icons.Default.Security) berwarna putih
- Typography: 
  - Nama App: 28sp, Bold, White
  - Subtitle: 16sp, Medium, White.copy(alpha=0.8f)
  - Versi: 12sp, Regular, White.copy(alpha=0.6f)
- Alignment: Semua elemen center secara vertical & horizontal

## STATE MANAGEMENT
- Gunakan `remember` untuk animasi alpha (Animatable)
- Gunakan `remember` untuk animasi scale (Animatable)
- LaunchedEffect(Unit) untuk trigger animasi & delay

## OUTPUT
Berikan kode Kotlin lengkap dengan:
1. Composable function `SplashScreen(navController: NavController)`
2. Preview function `@Preview`
3. Tidak ada hardcoded string, gunakan stringResource
```

### Prompt B: Login Screen

```markdown
## KONTEKS
Buat Login Screen untuk Admin Dashboard aplikasi Absensi Pro. 
Admin login menggunakan ID Karyawan + PIN 4 digit.

## TUGAS
Buatkan composable `LoginScreen` yang:
1. Header dengan tombol back (opsional) dan judul "Selamat Datang Admin! 👋"
2. Input field ID Karyawan (text field biasa)
3. Input field PIN 4 digit dengan visual dot (••••) bukan plaintext
4. Checkbox "Ingat Saya"
5. Tombol "MASUK" dengan full width
6. Link "Lupa PIN?"
7. Validasi form sebelum submit
8. Loading state saat API call

## BACKEND API
```json
POST {scriptUrl}
{
  "action": "login",
  "idKaryawan": "ADM001",
  "pin": "1234",
  "deviceId": "android-uuid",
  "deviceName": "Samsung A54",
  "force": false
}
```
Response sukses:
```json
{
  "success": true,
  "user": {
    "id": "ADM001",
    "nama": "Budi Admin",
    "jabatan": "Admin",
    "tokoDefault": "T001",
    "shiftDefault": "S001",
    "noHP": "08123456789",
    "email": "budi@email.com",
    "fotoProfil": "https://..."
  },
  "modeJadwal": "LOCKED"
}
```
Response gagal (device beda):
```json
{
  "success": false,
  "requireDeviceConfirmation": true,
  "message": "Akun Anda sudah login di perangkat: Samsung A12..."
}
```

## SPESIFIKASI UI
- Background: Color(0xFFF8F9FA)
- Card container dengan padding 24dp, corner 16dp, elevation 2dp
- Input field: OutlinedTextField dengan shape RoundedCornerShape(12.dp)
- PIN input: Numeric keyboard, max 4 karakter, visual transformation ke dot
- Tombol MASUK: height 52dp, background Color(0xFF1A73E8), text white bold
- Error: Snackbar merah dengan shake animation pada input

## STATE MANAGEMENT
Gunakan ViewModel pattern:
- `var idKaryawan by mutableStateOf("")`
- `var pin by mutableStateOf("")`
- `var isLoading by mutableStateOf(false)`
- `var errorMessage by mutableStateOf<String?>(null)`
- `var showDeviceDialog by mutableStateOf(false)`

## ERROR HANDLING
1. ID Karyawan kosong → error "ID Karyawan wajib diisi"
2. PIN kurang dari 4 → error "PIN harus 4 digit"
3. Network error → Snackbar "Koneksi terputus, coba lagi"
4. Response `requireDeviceConfirmation` → tampilkan AlertDialog konfirmasi
5. Login gagal → Snackbar dengan pesan dari API

## OUTPUT
1. `LoginScreen(navController: NavController, viewModel: LoginViewModel = hiltViewModel())`
2. `LoginViewModel` class lengkap
3. `DeviceConfirmationDialog` composable
4. Preview untuk semua state (default, loading, error)
```

## 2️⃣ LAMAN: DASHBOARD UTAMA (HOME)

### Prompt C: Dashboard Screen (Utama)

```markdown
## KONTEKS
Buat Dashboard Home Screen untuk Admin yang menampilkan ringkasan seluruh 
informasi absensi hari ini secara sekilas.

## TUGAS
Buatkan composable `DashboardScreen` yang terdiri dari beberapa section:

### SECTION 1: Header Profile
- Card dengan background gradient Primary → PrimaryDark
- Avatar karyawan (AsyncImage, 56dp, circle)
- Nama admin + tanggal hari ini (format: "Senin, 29 Mei 2026")
- Icon notifikasi dengan Badge (jumlah pending approval)

### SECTION 2: Stats Summary (4 card grid)
- Hadir: icon People, color Green, value dari API
- Telat: icon Schedule, color Yellow, value dari API  
- Izin: icon Description, color Blue, value dari API
- Cuti: icon Warning, color Red, value dari API

### SECTION 3: Monitoring Toko (Horizontal scroll)
- Card per toko dengan:
  - Foto toko (80dp, rounded 12dp)
  - Nama toko
  - Jam buka-tutup
  - Progress bar online (online/total karyawan)
  - Indikator jumlah yang telat (jika ada)

### SECTION 4: Aktivitas Terbaru (Vertical list, max 5 item)
- List absensi terbaru hari ini
- Status indicator (dot: green=ontime, yellow=telat, blue=pending)
- Nama karyawan, waktu, toko, status chip

### SECTION 5: Menu Cepat (Grid 2x4)
- Icon button: Karyawan, Toko, Jadwal, Laporan, Approval, Lembur, Chat, Settings

## BACKEND API
```json
// Dashboard Stats
POST {action: "getDashboardData"}
Response: {
  "success": true,
  "stats": {
    "totalKaryawan": 24,
    "hadirHariIni": 20,
    "pendingApproval": 5,
    "telatHariIni": 3
  }
}

// Monitoring Toko
POST {action: "getMonitoringToko"}
Response: {
  "success": true,
  "toko": [
    {
      "idToko": "T001",
      "namaToko": "Toko Pusat",
      "fotoUrl": "https://...",
      "jamBuka": "08:00",
      "jamTutup": "22:00",
      "totalKaryawan": 8,
      "totalOnline": 5,
      "karyawan": [
        {"nama": "Andi", "status": "hadir", "menitTelat": 0},
        {"nama": "Budi", "status": "telat", "menitTelat": 10}
      ]
    }
  ]
}
```

## SPESIFIKASI UI
- Pull-to-refresh menggunakan `rememberPullRefreshState`
- LazyColumn untuk scrollable content
- Card elevation: 2dp default, 4dp on press
- Shimmer loading untuk semua section saat initial load
- Empty state jika tidak ada data

## STATE MANAGEMENT
```kotlin
data class DashboardUiState(
    val isLoading: Boolean = false,
    val isRefreshing: Boolean = false,
    val stats: DashboardStats? = null,
    val tokoList: List<TokoMonitor> = emptyList(),
    val recentActivities: List<ActivityItem> = emptyList(),
    val error: String? = null
)
```

## REAL-TIME
- Subscribe Pusher channel "pinguin-chat", event "absen-alert"
- Saat ada event baru, tambahkan ke `recentActivities` (prepend, max 10)
- Update stats count secara real-time

## OUTPUT
1. `DashboardScreen` composable lengkap
2. `DashboardViewModel` dengan semua logic
3. Sub-composable: `HeaderProfileCard`, `StatsSummaryCard`, `TokoMonitoringCard`, `ActivityFeedItem`, `QuickMenuGrid`
4. Data class untuk semua model
5. Preview untuk loading, success, empty, error states
```
